A block-breaker game developed with Assembly(MASM32)

基于MASM32汇编开发的打砖块游戏

### 基本玩法

横屏打砖块游戏，中间生成3行5列的砖块，两侧为可上下移动的挡板。每个玩家各自有一个球。

左侧玩家：AD键控制。右侧玩家：上下方向键控制。

### 血量机制

#### 砖块血量
每个砖块血量开局时随机生成，1-5。被球打中时扣一滴血，为0时消失。

#### 玩家血量
每个玩家3滴血，一方死亡时游戏结束。目的是接住自己的球，躲避对方的球。

若自己的球没有接住（失误），则扣一滴血，球重置为初始状态。

若接住了对方的球，则扣一滴血，球正常回弹。

己方球碰撞对面无挡板墙壁，正常回弹。

### 染色机制
球、挡板和砖块均有5种颜色：红，深蓝，蓝绿，紫，浅蓝。

开局时各物体颜色随机。

己方挡板和球碰撞时，球变为挡板的颜色，而后挡板颜色随机更新。

己方挡板和敌方球碰撞，双方球都不变。

### 元素反应机制
只有球与砖块会触发元素反应。球与挡板、球与球均不会。

元素和颜色对应关系：

火，红色

水，深蓝色

风，蓝绿色

雷，紫色

冰，浅蓝色


一个球和一个砖块碰撞时：

固定血量减一。

若属性相同：无任何行为。


属性不同：

若一方为风，发生扩散反应：若球是风元素，把碰撞砖块邻近砖块的属性染成和碰撞砖块相同颜色；若砖块是风元素，把砖块邻近砖块的属性染成和球相同的颜色。


若是其它两种不同元素（水火冰雷）间反应，则不区分球或砖块。


水+火：蒸发。砖块直接消失。球的状态变为初始值。但是相应玩家不扣血。

水+冰：冻结。球停在碰撞的地方2秒钟不动，然后正常反弹。

水+雷：感电。场上所有水元素的方块血量-1。

火+冰：融化。场上所有冰元素方块变为水元素。

火+雷：超载。附近的砖块血量也-1。
